## 安装

在 [Releases](https://github.com/yavlenie-pro/prossh/releases) 页面选择适合你系统的安装包：

- **Windows** —— `ProSSH_<version>_x64-setup.exe`（NSIS，推荐）或 `.msi` 版本。两种安装包都已本地化为英文、俄文和简体中文：NSIS 版 `.exe` 会根据系统语言自动选择界面语言；`.msi` 提供三个独立文件（`_en-US.msi`、`_ru-RU.msi`、`_zh-CN.msi`）—— 请选择与你的语言匹配的版本。需要 [WebView2 运行时](https://developer.microsoft.com/microsoft-edge/webview2/)；Windows 11 已预装。
- **macOS** —— Apple Silicon 选择 `ProSSH_<version>_aarch64.dmg`，Intel 选择 `ProSSH_<version>_x64.dmg`。当前构建未签名；首次启动需要右键 → **打开** 以绕过 Gatekeeper。
- **Linux** —— Debian/Ubuntu 衍生版本用 `.deb`，或使用可移植的 `.AppImage`。需要 `libwebkit2gtk-4.1`（`.deb` 包会自动安装）和密码服务（`gnome-keyring` 或 `kwallet`）来存储密码。

## 首次启动和数据位置

ProSSH 首次启动时会创建两个目录：

| 用途 | Windows | macOS | Linux |
| --- | --- | --- | --- |
| SQLite 数据库、调试日志 | `%APPDATA%\prossh\` | `~/Library/Application Support/prossh/` | `~/.local/share/prossh/` |
| Known hosts、可编辑的 JSON | `%APPDATA%\prossh\`（同上） | `~/Library/Preferences/prossh/` | `~/.config/prossh/` |

凭据永远不会存储在数据库里——它们会进入操作系统的密钥链，服务名为 `prossh`（密钥口令使用单独的服务名 `prossh-keypass`）。

## 会话和分组

**会话**是一份保存好的连接配置：主机、端口、用户、身份验证方式、描述，以及可选的颜色。会话位于左侧边栏。

- **新建会话**——侧边栏 → `+` 按钮，或命令面板中的 **新建会话** 动作。
- **分组**——右键点击侧边栏 → **新建分组**。从会话的右键菜单 → **移动到分组**，将会话拖入分组。
- **搜索**——侧边栏的搜索框按名称、主机、用户名或描述过滤。
- **重命名**——在详情面板中双击会话名称，或使用其右键菜单。
- **复制**——右键菜单 → **复制**。适合将一个会话作为跳板模板复制一份。
- **删除**——右键菜单 → **删除**（会询问确认）。
- **去重**——设置 → 会话 → **移除重复项** 会删除具有相同 `host:port+user` 的会话，保留最近使用的那一个。

会话带有一个 **最近使用** 时间戳，用于决定默认排序，并显示在详情面板中。

## 连接

选择一个会话并按 **连接**（或在聚焦行上按 `Enter`）。ProSSH 将执行以下步骤：

1. 解析 DNS，打开 TCP 套接字（10 秒超时）。
2. 使用 `russh` 协商 SSH 协议（默认使用现代算法：chacha20-poly1305、AES-GCM、ed25519、curve25519）。
3. 对照 `known_hosts.json` 验证主机密钥。新密钥会触发 TOFU 提示；变更后的密钥会被拒绝并在终端中显示错误。
4. 使用配置好的方法进行身份验证（见 [身份验证](#身份验证)）。
5. 请求 PTY，打开 shell，将 stdout/stderr 连接到 xterm 面板。
6. 后台探测 `uname -s` / Windows 的 `ver` 以检测远端 OS 家族——纯视觉效果（在会话列表显示正确的图标）。

**测试**（在会话详情面板中）执行步骤 1-4 但不打开 PTY——这是验证凭据最快的方法。

## 身份验证

支持三种方法：

- **密码**——输入一次，可选保存到系统密钥链（勾选 **将密码保存到系统密钥链**）。如果不保存，ProSSH 每次连接都会询问。
- **私钥**——指定一个私钥文件（OpenSSH 或 PEM）。如果私钥已加密，ProSSH 会在连接时显示口令对话框；口令也可以保存到密钥链。
- **SSH agent**——将身份验证委托给运行中的 agent（Windows 上配合 PuTTY 用 `pageant`，macOS/Linux 用 `ssh-agent`，Windows 10+ 内置了一个）。

支持的密钥格式：`RSA`、`ECDSA`、`Ed25519`（OpenSSH 和 PEM 编码都支持）。如果服务器对你的 agent/密钥返回 `publickey` 拒绝，ProSSH 会退回到 **凭据** 对话框，你可以在那里输入密码。

## 主机密钥（TOFU）

首次连接新主机会弹出 **未知主机密钥** 对话框。对话框中显示：

- 算法（例如 `ssh-ed25519`）
- SHA-256 指纹（`SHA256:Qw…`）

在点击 **接受并信任** 之前，请通过可信的带外渠道核对指纹。拒绝会直接关闭连接，不会保存任何内容。

**不匹配**——如果服务器提供的密钥与你之前信任的不同，ProSSH 会拒绝连接（`Host key mismatch — refused`）并保留旧条目。要覆盖，打开 **设置 → Known hosts**，找到对应的 `host:port` 行，点击 **撤销**，然后重新连接。

**管理 known hosts**——设置 → **Known hosts** 列出所有受信任条目及添加日期。你可以撤销单个条目，也可以一次性清除某个主机的全部密钥。

## 终端、标签页和分屏

每个连接会在顶部标签栏作为一个 **标签页** 打开。点击标签标题切换，点击 `×` 关闭。中键单击也会关闭。

**分屏**——当一个终端标签页处于活动状态时：

- `Ctrl+Shift+D` 或命令面板 → **水平分屏**——在右侧增加一个面板。
- `Ctrl+Shift+E` 或命令面板 → **垂直分屏**——在下方增加一个面板。
- 点击任意面板聚焦它；活动面板会有细微的边框高亮。
- 拖动分割线改变大小。通过各自的 `×` 关闭面板;关闭最后一个面板会关闭标签页。

**终端控制**：

- 选中即复制——如果在设置中启用，任何选择都会自动复制到剪贴板。
- 右键粘贴——无需通过系统菜单即可从剪贴板粘贴。
- 搜索——在终端内 `Ctrl+F` 打开内联搜索。
- 重连——在断开的面板上按 `R` 重连会话。

**URL 识别**——终端输出中的 `http://` / `https://` / `ssh://` 链接可点击,会用默认浏览器/客户端打开。

## SFTP 文件浏览器

将侧边栏切换到 **文件**。选择一个会话会自动在同一连接上打开 SFTP 通道。

- 使用面包屑栏、`后退` / `前进` 箭头或直接输入路径来导航。
- **上传**——将文件从操作系统拖到面板上,或使用 **上传** 按钮。底部的传输队列显示进度和预计时间;点击 `×` 可取消单个传输。
- **下载**——右键文件 → **下载**。
- **重命名** / **删除** / **新建文件** / **新建目录** / **chmod**——右键菜单。
- **拖出**——将远程文件拖到原生 OS 窗口(Finder、资源管理器)即可将其下载到该位置。
- **打开**——右键 → **打开** 目录会进入该目录;对文件,**用默认程序打开** 会下载到临时文件并启动 OS 处理程序。
- **在标签页中打开 SFTP**——将侧面板变成全屏双栏视图。

## 在你的编辑器中编辑远程文件

右键任意远程文件 → **在编辑器中编辑**:

1. ProSSH 将文件下载到临时路径。
2. 在配置好的编辑器中打开(首次使用可浏览选择 `code`、`subl`、`micro` 等)。
3. 监视本地文件变更。
4. 每次保存时,将新内容上传回服务器(如果启用了 **自动**;否则在传输面板中点击 **上传**)。
5. 点击 **停止监视** 结束会话;临时文件会被删除。

大文件、二进制文件和目录不会被监视器处理。

## 服务器间拷贝

想把文件从服务器 A 搬到服务器 B,又不想经过你的笔记本中转? 在服务器 A 的 SFTP 面板右键文件/目录 → **拷贝到服务器……**

向导会依次引导:

1. **选择目标会话** 和目标路径。
2. ProSSH 检查 **源** 上是否安装了 `rsync` 或 `scp`。如果没有,会提供通过检测到的包管理器安装它。
3. ProSSH 在源上查找 `~/.ssh/id_ed25519`。如果没有,会提供生成一个。
4. ProSSH 将 **公** 钥上传到目标的 `authorized_keys`(会询问目标的密码)。
5. 传输在源服务器自身运行,实时进度会流式传输到向导中。
6. 可选的清理会在传输后移除源和目标上的临时密钥对。

对于大体量数据,这比 "先下载到本地 → 再上传到另一台服务器" 的模式要快得多,因为字节永远不会离开数据中心。

## 端口转发

每个会话的转发规则位于会话编辑器 → **端口转发** 区:

- **本地**——`-L` 方向。在你的机器上绑定 `bind_host:bind_port`;对该套接字的请求会从 SSH 服务器的视角隧道到 `target_host:target_port`。
- **远程**——`-R` 方向。在 SSH 服务器上绑定;转发回你的机器上能访问到的 `target_host:target_port`。

规则默认启用,每次连接时会自动激活。可以禁用单条规则(或删除它)而不影响会话连接。

## 脚本和命令面板

**脚本** 是一个带名字的 bash 片段。按 `Ctrl+Shift+P` 打开命令面板,输入脚本名称的一部分,按 `Enter`——脚本文本就会被键入当前终端面板。

- **全局脚本**——任何会话都可使用。在设置 → **脚本** 中管理。
- **按会话脚本**——绑定到某一个会话,当该会话聚焦时会显示在全局脚本上方。在会话编辑器 → **脚本** 标签中管理。

命令面板还会展示所有会话(连接 / SFTP 两种变体)、分屏动作、设置和主题切换。它是最快速的导航方式。

## 从其他客户端导入

侧边栏 `+` → **从……导入**(或命令面板搜索 "导入")。

- **SSH config**——解析 `~/.ssh/config`,保留 `HostName` / `Port` / `User` / `IdentityFile`。你可选择要导入哪些 `Host` 条目。
- **PuTTY(仅 Windows)**——从 `HKCU\Software\SimonTatham\PuTTY\Sessions` 读取保存的会话。
- **MobaXterm(仅 Windows)**——从注册表读取,或加载 `.mxtsessions` 导出文件。MobaXterm 中的文件夹可以(通过切换复选框)变成 ProSSH 中的分组。

如果没有指定密钥,导入的会话默认使用 **密码** 验证。首次连接时会询问密码(并提供保存到密钥链的选项)。

## 配色方案

设置 → **外观** → **配色方案**。ProSSH 自带了几个内置主题:

- Windows Terminal Campbell / One Half Dark / One Half Light
- Solarized Dark / Solarized Light
- Dracula、Nord、Gruvbox、Tokyo Night

克隆任意内置主题,在编辑器中修改颜色后保存,即可创建自定义配色。内置主题不能直接修改(编辑会创建副本)。

配色变更仅对 **新打开** 的终端生效;已有的面板会保留打开时的颜色,直到关闭。

## 故障排查

**应用启动不了 / 看不到窗口(Windows)。** 检查 WebView2 运行时是否已安装。Windows 11 已预装;Windows 10 请从 [微软官网](https://developer.microsoft.com/microsoft-edge/webview2/) 获取。

**"Connection to host:port timed out (10s)"** —— TCP 握手未完成。可能是防火墙、VPN、端口错误,或服务器宕机。在终端里执行 `ping host` / `nc -vz host port` 以缩小范围。

**"Public key rejected by server"** —— 服务器的 `authorized_keys` 中没有匹配的公钥。用 `ssh-keygen -y -f /path/to/key` 验证你持有正确的私钥,并将 `/path/to/key.pub` 追加到远程的 `~/.ssh/authorized_keys`。

**"Host key mismatch — refused"** —— 自上次信任以来,服务器的指纹变了。合法情况是服务器 OS 重装,非法情况是发生了 MITM。设置 → Known hosts → 通过带外渠道验证新指纹后,**撤销** 旧条目。

**"No password stored for this session"** —— 你取消勾选了 *保存密码*。打开会话编辑器重新保存密码,或切换到密钥验证。

**连接时 SFTP 卡死,但终端正常。** 有些服务器对 SFTP 子系统有按用户的限制。检查 `sshd_config` 中的 `Subsystem sftp` 和 `Match User` 块。

**Linux 上粘贴缓慢 / 剪贴板失败。** 确保已安装 `xclip` 或 `wl-clipboard`(xterm.js 通过 Tauri 将剪贴板操作委托给系统)。

**桌面调试日志。** Windows 上位于 `%USERPROFILE%\Desktop\prossh-debug.log`(其他平台在系统桌面),包含早期启动诊断和解析事件跟踪。提交 bug 时请附上。

仍然卡住? 请在 <https://github.com/yavlenie-pro/prossh/issues> 开 issue,并附上:

- OS 名称和版本(`winver`、`sw_vers`、`uname -a`)
- ProSSH 版本(关于对话框)
- 重现步骤
- `prossh-debug.log`(如果有)
