## Установка

Возьмите артефакт под свою ОС со страницы [Releases](https://github.com/yavlenie-pro/prossh/releases):

- **Windows** — `ProSSH_<версия>_x64-setup.exe` (NSIS, рекомендуется) или вариант `.msi`. Оба инсталлятора локализованы на английский, русский и упрощённый китайский: NSIS-`.exe` автоматически выбирает язык по локали ОС; `.msi` поставляется тремя отдельными файлами (`_en-US.msi`, `_ru-RU.msi`, `_zh-CN.msi`) — берите соответствующий вашему языку. Требуется [WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/); в Windows 11 он предустановлен.
- **macOS** — `ProSSH_<версия>_aarch64.dmg` для Apple Silicon, `ProSSH_<версия>_x64.dmg` для Intel. Сборки пока без подписи; для первого запуска Gatekeeper потребует правого клика → **Open**.
- **Linux** — `.deb` для Debian/Ubuntu или портативный `.AppImage`. Нужен `libwebkit2gtk-4.1` (ставится автоматически из `.deb`) и Secret Service (`gnome-keyring` или `kwallet`) для хранения паролей.

## Первый запуск и расположение данных

При первом запуске ProSSH создаёт два каталога:

| Назначение | Windows | macOS | Linux |
| --- | --- | --- | --- |
| SQLite-база, debug-лог | `%APPDATA%\prossh\` | `~/Library/Application Support/prossh/` | `~/.local/share/prossh/` |
| known_hosts, правимый JSON | `%APPDATA%\prossh\` (тот же) | `~/Library/Preferences/prossh/` | `~/.config/prossh/` |

Учётные данные не пишутся в базу — они уходят в OS keychain под именем сервиса `prossh` (и `prossh-keypass` для passphrase'ов ключей).

## Сессии и группы

**Сессия** — сохранённый профиль подключения: host, порт, пользователь, метод аутентификации, описание, опциональный цвет. Сессии живут в левом sidebar.

- **Новая сессия** — кнопка `+` в sidebar или действие **New session** в command palette.
- **Группы** — правый клик по sidebar → **New group**. Перетащите сессии в группу через контекстное меню → **Move to group**.
- **Поиск** — строка поиска в sidebar фильтрует по имени, host, username, описанию.
- **Переименование** — двойной клик по имени в панели деталей, либо контекстное меню.
- **Дублирование** — контекстное меню → **Duplicate**. Удобно, чтобы клонировать сессию как шаблон для jump-host.
- **Удаление** — контекстное меню → **Delete** (с подтверждением).
- **Дедупликация** — Settings → Sessions → **Remove duplicates** удаляет сессии с одинаковыми `host:port+user`, оставляя самую свежую по last-used.

У каждой сессии есть timestamp **last used** — он влияет на сортировку по умолчанию и отображается в панели деталей.

## Подключение

Выделите сессию и нажмите **Connect** (или `Enter` на строке сессии). ProSSH:

1. Резолвит DNS и открывает TCP-сокет (timeout 10 с).
2. Договаривается по SSH с помощью `russh` (современные шифры по умолчанию: chacha20-poly1305, AES-GCM, ed25519, curve25519).
3. Сверяет host key с `known_hosts.json`. Новый ключ показывает TOFU-диалог; изменившийся — отклоняет с ошибкой в терминале.
4. Аутентифицируется выбранным методом (см. [Аутентификация](#аутентификация)).
5. Запрашивает PTY, открывает shell, пробрасывает stdout/stderr в xterm-панель.
6. В фоне выполняет `uname -s` / Windows `ver`, чтобы определить семейство ОС — это чисто для иконки в списке сессий.

**Test** (в панели деталей) выполняет шаги 1-4 без открытия PTY — быстрейший способ проверить учётку.

## Аутентификация

Три метода:

- **Password** — вводится один раз, опционально сохраняется в OS keychain (чекбокс `Store password in OS keychain`). Если не сохранять, ProSSH будет спрашивать каждый connect.
- **Private key** — путь к файлу приватного ключа (OpenSSH или PEM). Если ключ зашифрован, ProSSH покажет диалог passphrase при подключении; его тоже можно сохранить в keychain.
- **SSH agent** — делегирует аутентификацию запущенному агенту (`pageant` на Windows с PuTTY, `ssh-agent` на macOS/Linux, Windows 10+ имеет встроенный).

Поддерживаются ключи `RSA`, `ECDSA`, `Ed25519` (OpenSSH и PEM). Если сервер отклонил публичный ключ, ProSSH покажет диалог **Credentials**, в котором можно ввести пароль.

## Host keys (TOFU)

Первое подключение к новому хосту → диалог **Unknown host key** с:

- Алгоритмом (например, `ssh-ed25519`)
- SHA-256 fingerprint (`SHA256:Qw…`)

Перед **Accept & Trust** сверьте fingerprint через доверенный канал (страница хостера, админ сервера). Reject просто закроет соединение без сохранения.

**Mismatch** — если сервер показал ключ, отличающийся от того, что вы принимали ранее, ProSSH отклонит соединение (`Host key mismatch — refused`) и сохранит старую запись. Чтобы подтвердить новый ключ — Settings → Known hosts, найдите строку `host:port`, нажмите **Revoke** и подключитесь снова.

**Управление known hosts** — Settings → **Known hosts** показывает все доверенные записи с датой добавления. Можно отозвать одну запись или все ключи хоста целиком.

## Терминал, вкладки и split-панели

Каждое подключение — отдельная **вкладка** в верхней панели. Клик по лейблу переключает, `×` закрывает. Средняя кнопка мыши тоже закрывает.

**Split panes** — с активной вкладкой терминала:

- `Ctrl+Shift+D` или palette → **Split pane horizontally** — добавляет панель справа.
- `Ctrl+Shift+E` или palette → **Split pane vertically** — добавляет панель снизу.
- Клик по панели фокусирует её; у активной — подсветка рамкой.
- Перетащите разделитель, чтобы изменить размер. `×` закрывает одну панель; закрытие последней закрывает вкладку.

**Управление терминалом**:

- Copy on select — если включено в настройках, выделение автоматически копируется в буфер обмена.
- Right-click paste — вставка из буфера без системного меню.
- Поиск — `Ctrl+F` внутри терминала открывает инлайн-поиск.
- Reconnect — `R` на отключённой панели переподключает сессию.

**Распознавание ссылок** — `http://` / `https://` / `ssh://` в выводе терминала кликабельны и открываются в браузере или клиенте по умолчанию.

## SFTP-менеджер

Переключите sidebar на **Files**. Выбор сессии автоматически открывает SFTP-канал поверх того же соединения.

- Навигация — breadcrumb, `Back` / `Forward` или ввод пути вручную.
- **Upload** — перетащите файлы из ОС в панель или нажмите **Upload**. Очередь передач внизу показывает прогресс и ETA; `×` отменяет отдельные передачи.
- **Download** — правый клик по файлу → **Download**.
- **Rename** / **Delete** / **New file** / **New directory** / **chmod** — контекстное меню.
- **Drop out** — перетащите удалённый файл в нативное окно ОС (Finder, Explorer), чтобы скачать в эту папку.
- **Open** — правый клик по директории открывает её; на файле **Open in default app** скачивает во временный файл и запускает системный обработчик.
- **Open SFTP in tab** — разворачивает боковую панель в полноэкранный двухпанельный вид.

## Редактирование удалённых файлов в своём редакторе

Правый клик на удалённом файле → **Edit in editor**:

1. ProSSH скачивает файл во временный путь.
2. Открывает его в настроенном редакторе (первый раз — диалог выбора `code`, `subl`, `micro` и т.п.).
3. Следит за локальным файлом.
4. На каждом сохранении заливает новый контент обратно на сервер (если включён **Auto**; иначе — кнопка **Upload** в панели Transfers).
5. **Stop watching** завершает сессию; временный файл удаляется.

Большие файлы, бинарники и директории исключены из watcher'а.

## Server-to-server копирование

Нужно переложить файл с сервера A на сервер B, не гоняя его через свой ноутбук? Правый клик на файле/папке в SFTP-панели сервера A → **Copy to server…**

Мастер пошагово:

1. **Выбор destination-сессии** и целевого пути.
2. ProSSH проверяет, есть ли на **источнике** `rsync` или `scp`. Если нет — предлагает установить (через определённый пакет-менеджер).
3. ProSSH ищет `~/.ssh/id_ed25519` на источнике. Если нет — предлагает сгенерировать.
4. Заливает **публичный** ключ в `authorized_keys` на destination (спрашивая его пароль).
5. Копирование выполняется на самом сервере-источнике, с живым прогрессом в окне мастера.
6. Опциональная чистка — удаляет временную пару ключей с источника и destination после передачи.

Это драматически быстрее чем «скачать локально → залить на другой сервер» для больших объёмов — байты вообще не покидают датацентр.

## Port forwarding

Правила проброса на каждую сессию живут в редакторе сессии → секция **Port Forwarding**:

- **Local** — направление `-L`. Слушает `bind_host:bind_port` на вашей машине; запросы туда туннелируются в `target_host:target_port` с точки зрения SSH-сервера.
- **Remote** — направление `-R`. Слушает на SSH-сервере; туннелирует обратно в `target_host:target_port`, доступный с вашей машины.

Правила по умолчанию включены и активируются автоматически на каждом connect. Отдельные правила можно отключить (или удалить), не разрывая сессию.

## Скрипты и command palette

**Скрипт** — bash-сниппет с именем. `Ctrl+Shift+P` открывает command palette, наберите часть имени скрипта, `Enter` — текст набивается в текущую панель терминала.

- **Global scripts** — доступны из любой сессии. Управляются в Settings → **Scripts**.
- **Per-session scripts** — привязаны к одной сессии, показываются выше глобальных, когда эта сессия в фокусе. Управляются в редакторе сессии → вкладка **Scripts**.

Palette также показывает все сессии (варианты connect / SFTP), split-действия, настройки и переключение темы. Это самый быстрый способ перемещаться по приложению.

## Импорт из других клиентов

`+` в sidebar → **Import from…** (или поиск "Import" в palette).

- **SSH config** — парсит `~/.ssh/config`, учитывает `HostName` / `Port` / `User` / `IdentityFile`. Выбираете, какие записи `Host` импортировать.
- **PuTTY (только Windows)** — читает сохранённые сессии из `HKCU\Software\SimonTatham\PuTTY\Sessions`.
- **MobaXterm (только Windows)** — читает из реестра или загружает `.mxtsessions`-файл. Папки из MobaXterm становятся группами в ProSSH (переключается чекбоксом).

Импортированные сессии по умолчанию используют **password**-auth, если ключ не указан. Пароль запросится при первом connect (с предложением сохранить в keychain).

## Цветовые профили

Settings → **Appearance** → **Color Profile**. В комплекте несколько встроенных тем:

- Windows Terminal Campbell / One Half Dark / One Half Light
- Solarized Dark / Solarized Light
- Dracula, Nord, Gruvbox, Tokyo Night

Свой профиль создаётся клонированием любого встроенного, правкой цветов в редакторе и сохранением. Встроенные профили нельзя менять напрямую (правка создаёт копию).

Смена профиля применяется к **вновь открытым** терминалам; существующие панели сохраняют цвет, с которым открылись, до закрытия.

## Траблшутинг

**Приложение не запускается / окно не появляется (Windows).**  Проверьте установлен ли WebView2 Runtime. На Windows 11 он предустановлен; на Windows 10 скачайте с [microsoft.com](https://developer.microsoft.com/microsoft-edge/webview2/).

**"Connection to host:port timed out (10s)"** — TCP handshake не завершился. Файрвол, VPN, неправильный порт, либо сервер down. Проверьте `ping host` / `nc -vz host port` из терминала.

**"Public key rejected by server"** — в `authorized_keys` на сервере нет подходящего публичного ключа. Проверьте `ssh-keygen -y -f /path/to/key` что у вас правильный приватный ключ и допишите `/path/to/key.pub` в `~/.ssh/authorized_keys` на сервере.

**"Host key mismatch — refused"** — fingerprint сервера изменился с последнего раза, когда вы ему доверяли. Это легитимно при переустановке ОС сервера и подозрительно при MITM. Settings → Known hosts → **Revoke** устаревшую запись, предварительно сверив новый fingerprint через доверенный канал.

**"No password stored for this session"** — вы сняли галочку *Store password*. Откройте редактор сессии и сохраните пароль снова, либо переключитесь на key auth.

**SFTP зависает на connect, а терминал работает.**  Некоторые серверы ограничивают SFTP subsystem по пользователю. Проверьте `sshd_config` — блоки `Subsystem sftp` и `Match User`.

**Медленная вставка / проблемы с буфером обмена в Linux.**  Убедитесь, что `xclip` или `wl-clipboard` установлен (xterm.js делегирует буфер обмена системе через Tauri).

**Debug-лог на рабочем столе.**  `%USERPROFILE%\Desktop\prossh-debug.log` на Windows (и desktop на других ОС) содержит диагностику early-boot и трассу resolve-событий. Прикладывайте при баг-репортах.

Всё равно не запускается? Откройте issue на <https://github.com/yavlenie-pro/prossh/issues> и приложите:

- Имя и версию ОС (`winver`, `sw_vers`, `uname -a`)
- Версию ProSSH (диалог About)
- Шаги для воспроизведения
- `prossh-debug.log`, если есть
